# Owen Crabbe

Builder working at the intersection of human cognition and AI agents. Studying psychology and neuroscience at Ball State University. I design agent tooling and run small, reproducible research projects, and I care more about what survives testing than what sounds good.

Most of what I build answers one question. Can an AI agent do real, disciplined work instead of producing fast and plausible output. That standard runs through everything here.

### Selected work

**[agentic-quant-lab](https://github.com/owencrabbe/agentic-quant-lab)**
A reproducible research desk for an AI-agent-run trading process. Every result regenerates from the scripts on free data. The headline finding corrected my own assumption. Across 55 sessions and 6 tickers, the entry pattern was not the edge. A daily-trend regime filter turned a blind opening-range breakout from negative 0.06R to positive 0.21R expectancy and cut max drawdown from 28.7R to 4.8R. Ships with honest caveats about sample size and untested regimes. Python, MIT.

**[MindForge-Skills](https://github.com/owencrabbe/MindForge-Skills)**
A Claude Code plugin. Six reasoning skills that install structured thinking methods: research, first-principles, red-teaming, decision science, Socratic inquiry, and cross-disciplinary transfer. Built on one rule. Honesty over confidence, and end with something usable. MIT.

**[pago-skills](https://github.com/owencrabbe/pago-skills)**
Free, portable AI skills for everyday and real-estate tasks, written in plain language so they run on any capable agent. The open companion to PAGO, a city intelligence platform. MIT.

### How I work

- Evidence over folklore. If a claim can be measured, I measure it and keep only what the data supports.
- Honest about uncertainty. Every project flags what is untested and what could break it.
- Ship, then sharpen. Real repos, reproducible steps, no vaporware.
